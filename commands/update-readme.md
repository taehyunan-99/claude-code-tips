# Update README

학습 레포지토리의 README.md를 아래 템플릿과 규칙에 따라 업데이트합니다.

## README 템플릿 구조

아래 템플릿을 기반으로 README를 작성/업데이트합니다.

```markdown
# 📚 [Study Repository Name]

[학습 분야] 학습을 위한 레포지토리입니다.

## 📂 폴더 구조
```
project/
├── docs/              # 학습 노트 및 정리
│   ├── 01_topic1.md
│   ├── 02_topic2.md
│   └── images/
├── examples/          # 예제 코드
│   ├── basic/
│   │   ├── example1.py
│   │   └── example2.py
│   └── advanced/
├── exercises/         # 연습 문제 및 실습
│   ├── problem1.py
│   ├── problem2.py
│   └── solutions/
├── projects/          # 실제 프로젝트
│   ├── project1/
│   └── project2/
├── resources/         # 참고 자료
│   ├── references.md
│   └── links.md
└── README.md          # 프로젝트 개요
```

- **세부 주제1**: 학습 내용 설명
- **세부 주제2**: 학습 내용 설명

<br/>

### 🎨 [기술명2]

<br/>

#### [세부 분야2]

<br/>

### ⚡ [기술명3]
```

## 작업 단계

### 1. 변경 파일 확인
- 새로 추가된 파일 식별
- 수정된 파일 확인
- 변경 사항 분석

### 2. 기존 README 분석
- 기존 README.md 구조 파악
- 섹션 형식, 들여쓰기, 스타일 확인
- 이모지 사용 패턴 파악

### 3. 내용 업데이트
- **폴더 구조**: 새 파일/폴더 추가 시 업데이트
- **학습 내용**: 변경된 파일의 실제 내용만 확인하여 추가
- 대화 중 설명만 한 내용은 제외
- 기존 README의 섹션 형식, 들여쓰기, 스타일 유지

### 4. 마무리
- 마지막 업데이트 날짜를 현재 날짜로 갱신 (YYYY-MM-DD)
- 이모지 규칙 준수

## 필수 준수 사항

### ✅ 포함해야 할 것
- 새로 추가되거나 수정된 파일의 실제 내용
- 파일에 실제로 존재하는 코드와 기능
- 기존 README의 구조와 스타일

### ❌ 제외해야 할 것
- 대화 중 설명만 하고 파일에 없는 내용
- 변경되지 않은 기존 내용의 수정
- 기존 README 구조 변경

## 이모지 사용 규칙

### 고정 이모지 (모든 README에서 동일)
- **제목**: 📚
- **폴더 구조**: 📂
- **학습 내용**: 📂

### 기술명별 이모지 (각 챕터 특성에 맞게)
- 각 기술/주제에 어울리는 이모지를 상황에 맞게 선택
- 예시:
  - HTML: 🏷️
  - CSS: 🎨
  - JavaScript: ⚡
  - SQL/Database: 🗄️
  - Python: 🐍
  - API/Backend: 🔧
  - Security: 🔐
- README 생성 시 해당 기술의 특성을 반영한 적절한 이모지 자동 적용

## 실행 방법

사용자가 `/update-readme`를 입력하면 자동으로 이 규칙에 따라 README.md를 업데이트합니다.
